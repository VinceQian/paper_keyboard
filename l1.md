# 四、逐分钟课案

## 0:00 - 0:05 开场：今天做什么

你可以直接说：

> 我们这个项目叫 Paper Keyboard，也就是纸面键盘。
> 最终目标是：用户在纸上敲一个按键，电脑能判断出他按了哪个键，并输出对应字符。
> 但是电脑本身看不懂“我按了纸上的 1”。
> 所以我们需要把真实世界里的动作，转换成程序能处理的数据。

无边记写：

```text
电脑不能直接理解：
“我按了纸上的哪个键”

我们要做的是：
真实动作 → 数据 → 判断 → 输出
```

然后说：

> 今天我们先不急着写完整程序。
> 我们先弄清楚这个系统里每一部分负责什么。

---

## 0:05 - 0:10 学生猜想：电脑需要知道什么？

无边记问题：

```text
如果让电脑判断“我按了哪个纸面按键”，它至少需要知道什么？
```

你问学生：

> 你想一下，如果电脑要判断你按了哪个键，它至少得知道哪些信息？

引导答案：

```text
1. 手指在哪里
2. 有没有真的敲下去
3. 每个按键在哪里
```

如果学生答不出来，你可以逐步问：

> 如果电脑不知道你的手指在哪里，它能判断吗？
> 如果你只是把手指放在键上，没有敲，它要不要输出？
> 如果电脑不知道纸上每个键的位置，它能知道你按的是 1 还是 2 吗？

无边记最终补全：

```text
手指在哪里 → fingers 里的 x, y
有没有敲击 → tap.audio
每个按键在哪里 → layout JSON
```

标准答案：

```text
电脑至少需要知道：
1. 手指坐标
2. 是否发生敲击
3. 按键布局
```

## 0:25 - 0:35 项目整体数据流

无边记写：

```text
真实世界：
用户在纸面键盘上敲击

摄像头：
找到手指在纸面上的位置

麦克风：
判断有没有敲击声音

session JSON：
把每一帧的数据保存下来

layout JSON：
记录每个按键在纸上的位置

InputEngine：
根据 frame 和 layout 判断是否输出字符
```

你可以这样讲：

> 这个项目可以拆成几个部分。
> 摄像头负责看手指在哪里。
> 麦克风负责听有没有敲击。
> session JSON 负责记录每一帧发生了什么。
> layout JSON 负责记录每个按键在哪里。
> InputEngine 是真正做判断的地方。

然后强调：

> 摄像头不直接告诉我们“按了 1”。
> 麦克风也不直接告诉我们“按了 1”。
> 摄像头只提供手指位置。
> 麦克风只提供有没有敲击。
> 真正判断按了哪个键的是 InputEngine。

无边记重点句：

```text
摄像头不直接判断按键。
麦克风不直接判断按键。
真正做判断的是 InputEngine。
```

# 五、session JSON 部分

## 0:45 - 0:50 引入 session

你可以说：

> 接下来我们先看 session JSON。
> session 可以理解成“一次输入过程的记录”。
> 比如我敲了 1、2、3，这整个过程就可以保存成一个 session 文件。

无边记写：

```text
session JSON：
记录一次输入过程

一个 session 里有很多 frame
每个 frame 表示某一个时刻发生了什么
```

然后说：

> 我们的程序不是直接处理“我按了 123”这句话。
> 它处理的是一帧一帧的数据。

---

## 0:50 - 0:58 讲 frame 字段

无边记放简化版字段，不要一上来放完整嵌套 JSON：

```text
一个 frame 里最重要的信息：

frame_id: 2
t: 0.033

fingers:
  finger_id: 1
  x: 106.0
  y: 82.8

tap:
  audio: true
```

你逐个解释：

```text
frame_id：第几帧
t：这一帧发生的时间
fingers：这一帧看到的手指
finger_id：手指编号
x, y：手指在纸面上的位置
tap.audio：这一帧有没有检测到敲击声
```

你可以说：

> x 和 y 不是屏幕坐标，而是纸面上的坐标。
> 我们最终希望 InputEngine 不关心摄像头画面，只关心纸面坐标。
> 所以摄像头那部分以后会负责把图像里的手指位置转换成纸面上的 x、y。

---

## 0:58 - 1:03 解释 finger_id = 1

无边记写：

```text
当前版本简化：

每一帧最多只有一个 finger
finger_id 固定为 1
1 代表食指

注意：
这不是系统现在已经能识别出食指
而是我们当前阶段的人为约定
```

你可以这样讲：

> 现在我们还没有做双摄像头，也没有做颜色贴纸识别，所以系统暂时不能真正区分每根手指。
> 为了先把主流程做通，我们先假设只有一个手指，也就是食指。
> 所以 finger_id 固定为 1。
> 以后如果我们能识别多个手指，这里就可以出现 finger_id = 1、2、3 等不同编号。

互动问题：

```text
finger_id = 1 是什么意思？
```

标准答案：

```text
当前版本里，finger_id = 1 表示我们假设正在使用的那个食指。
它不是系统真实识别出来的结果，而是当前阶段的简化约定。
```

# 六、layout JSON 部分

## 1:10 - 1:15 引入 layout

你可以说：

> 现在我们知道手指在哪里了。
> 但是光知道手指坐标还不够。
> 比如手指在 x=25, y=30，电脑并不知道这是按键 1、按键 2，还是空白区域。
> 所以我们还需要 layout。
> layout 记录每个按键在纸面上的位置。

无边记写：

```text
layout JSON：
记录纸面键盘上每个按键的位置

session 告诉我们：手指在哪里
layout 告诉我们：每个键在哪里
```

---

## 1:15 - 1:23 讲 key 的字段

无边记放：

```text
一个 key 的例子：

label: "1"
x: 20
y: 20
w: 20
h: 22
```

解释：

```text
label：这个键输出什么字符
x：按键左上角的 x 坐标
y：按键左上角的 y 坐标
w：按键宽度
h：按键高度
```

你可以说：

> 每个按键都可以看作一个矩形。
> x 和 y 是左上角的位置。
> w 是宽度，h 是高度。
> 所以这个按键覆盖的范围是：
> x 从 20 到 40，y 从 20 到 42。

无边记写：

```text
key 1 的范围：

x: 20 到 40
y: 20 到 42
```

---

## 1:23 - 1:30 点在不在矩形里

无边记写：

```text
判断一个 finger 是否在 key 里面：

finger_x >= key_x
finger_x <= key_x + key_w
finger_y >= key_y
finger_y <= key_y + key_h
```

你可以说：

> 这一步不是 AI，也不是很复杂的算法。
> 它本质上就是判断一个点有没有落在一个矩形范围里。
> 我们后面写 InputEngine 的时候，最核心的一步就是这个判断。

互动题：

```text
key 1:
x = 20
y = 20
w = 20
h = 22

点 A: x=25, y=30
点 B: x=45, y=30
点 C: x=25, y=50

它们在不在 key 1 里？
```

标准答案：

```text
A 在。
因为 25 在 20 到 40 之间，30 在 20 到 42 之间。

B 不在。
因为 x=45 超出了 20 到 40 的范围。

C 不在。
因为 y=50 超出了 20 到 42 的范围。
```

# 七、InputEngine 部分

## 1:35 - 1:42 InputEngine 是什么

无边记写：

```text
InputEngine：系统的大脑

它不直接处理摄像头画面。
它不直接处理麦克风声音。

它只读取：
1. frame 数据
2. layout 数据

然后判断：
这一帧要不要输出一个按键？
```

你可以说：

> InputEngine 是我们线上课最核心要写的部分。
> 它不需要知道摄像头怎么识别手指，也不需要知道麦克风怎么检测声音。
> 它只需要处理整理好的 frame 数据。
> 这样我们就可以先把核心判断逻辑学明白。

---

## 1:42 - 1:47 有效输入的两个条件

无边记写：

```text
一次有效输入需要两个条件：

条件 1：
手指在某个 key 里面

条件 2：
tap.audio 是 true
```

你可以讲：

> 这两个条件缺一不可。
> 如果手指在 key 上，但是没有敲击，不输出。
> 如果有敲击，但是手指不在任何 key 上，也不输出。
> 只有手指在某个 key 里，并且 audio=true，才输出这个 key。

互动题：

```text
情况 1：
手指在 key 1 上，audio=false
会不会输出？

情况 2：
手指在空白区域，audio=true
会不会输出？

情况 3：
手指在 key 1 上，audio=true
会不会输出？
```

标准答案：

```text
情况 1：不输出。
原因：没有检测到敲击。

情况 2：不输出。
原因：手指不在任何按键上。

情况 3：输出 key 1。
原因：手指在 key 上，并且检测到敲击。
```

# 八、Python 最小基础部分

## 1:50 - 1:55 dict / list 快速过一下

如果学生特别弱，这部分可以多讲。
如果学生已经理解，就快速带过。

无边记写：

```text
今天需要知道的 Python 基础：

dict：用名字取数据
list：用编号取数据
True / False：表示真假
print：输出结果
```

例子：

```text
frame["frame_id"]
frame["tap"]["audio"]
frame["fingers"][0]
finger["x"]
finger["y"]
```

你可以讲：

> JSON 读进 Python 以后，基本上就是 dict 和 list。
> dict 用名字取数据，比如 frame["tap"]。
> list 用编号取数据，比如 fingers[0]。
> 因为现在我们假设只有一个 finger，所以先取 fingers[0]。

互动问题：

```text
frame["tap"]["audio"] 取到的是什么？
```

答案：

```text
这一帧有没有检测到敲击声，结果是 true 或 false。
```

问题：

```text
frame["fingers"][0] 取到的是什么？
```

答案：

```text
这一帧里的第一个 finger。当前版本里，也就是我们假设的食指。
```

```text
session = ________
InputEngine = ________
tap.audio = ________
finger_id = 1 表示 ________
```

标准答案：

```text
layout = 按键在哪里
session = 每一帧发生了什么
InputEngine = 判断是否输出字符
tap.audio = 有没有检测到敲击
finger_id = 1 表示当前版本假设使用的食指
```

最后总结：

> 今天我们没有写很多代码，但我们把项目最重要的数据流讲清楚了。
> 后面写代码的时候，所有代码其实都是围绕这句话展开：
> 读取一帧 frame，看手指坐标在哪个 key 里，再看 audio 是不是 true，最后决定要不要输出字符。

# 十三、第一节课无边记核心笔记汇总

你可以直接把这些作为无边记里的重点卡片。

```text
电脑不能直接理解：
“我按了纸上的哪个键”

系统至少需要知道：
1. 手指在哪里
2. 有没有敲击
3. 每个按键在哪里
```

```text
摄像头：
提供手指位置

麦克风：
提供敲击信息

session JSON：
记录每一帧发生了什么

layout JSON：
记录每个按键在哪里

InputEngine：
判断是否输出字符
```

```text
session 里的一个 frame 主要包含：

frame_id：第几帧
t：时间
fingers：这一帧看到的手指
finger_id：手指编号
x, y：手指在纸面上的位置
tap.audio：有没有检测到敲击
```

```text
当前版本简化：

只有一个 finger
finger_id 固定为 1
1 代表食指

之后可以扩展：
双摄像头 / 颜色贴纸 → 多个 finger_id
```

```text
layout 里的一个 key 主要包含：

label：按键输出字符
x：按键左上角 x 坐标
y：按键左上角 y 坐标
w：按键宽度
h：按键高度
```

```text
判断 finger 是否在 key 里：

finger_x 在 key_x 到 key_x + key_w 之间
finger_y 在 key_y 到 key_y + key_h 之间
```

```text
一次有效输入：

手指在某个 key 里面
并且
tap.audio = true
```

```text
三种情况：

手指在 key 上，audio=false → 不输出
手指在空白处，audio=true → 不输出
手指在 key 上，audio=true → 输出
```

# 十四、课后小作业

建议第一节课作业不要难。

可以布置：

```text
作业 1：

作业 2：
回答下面 5 个问题：
```
